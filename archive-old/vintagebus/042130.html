Actually, to be speific and technical, there are two ways to do this. 

In the zone file for the domain they need to set EITHER an A record or a
CNAME for the _at_ portion of the zone.

IF they set an A record, the record must point to an IP address.  Then the
web server that corresponds to that IP must be configured to respond
correctly. (if the web server uses a seperate IP address for each website,
no further configuration should be required.  but if the server uses host
headers then a second (or even a wildcard..) host header should be
defined that will match on "domain_dot_com" in addition to "www.domain_dot_com")


Alternatively one may set a CNAME instead of an A record.  This is
essentially an alias, and has to point to a FQDN (fully qualified domain
name).  so for the _at_ section of the zone, they set a CNAME record that
will point to the www A record.

This will require an additional step when your resolver tries to find an
IP to associate to the name, but the webserver will see the same
www.domain_dot_com host header that it saw previously...
