It seems there are several ways to deal with the problem of web address
harvesting. You mention converting the addresses to .gif files which I've
never heard of before. There's also:

--A wpoison script. This is a script that generates an infinite number of
bogus email addresses. You can't get to the real addresses until you get
past this. It's not visible but you can play with it here:

	http://www.type2_dot_com/cgi-local/threebagsfull.pl

--Javascript obfuscation. This is endorsed by no less than Steve Linford of
Spamhaus Project fame, who claims it has not failed him since 1997 when he
started using it. For the address <support_at_uxn_dot_com>:

	<script language="JavaScript1.2"><!--
	//Anti-spam address builder
	var user   = "support"
	var domain = "uxn"
	var tld    = "com"
	document.write("<a href=" + "mail" + "to:" + user + "_at_" +
	domain + "." + tld + ">" + user + "_at_" + domain + "." + tld + "</a>")
	// -->
	</script>

--Hexadecimal obfuscation. This is when you encode the addresses in hex,
which can easily be done with resources such as
http://www.paradigm.nu/ascii2hex/ascii2hex.htm

	In hex, the address <address_at_bus_dot_com> is
	%61%64%64%72%65%73%73%40%62%75%73%2E%63%6F%6D

This will be handled seamlessly by a browser but it is useless in ASCII and
it has no atsign so a trawler won't see it. Long and short, spammers won't
be able to deal with it.
