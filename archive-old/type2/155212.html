grief.  

All of our servers pull their time from an outside source.  So that's no
problem.  

The cron jobs... well, the only things running in that time window are
system tasks.

but backups.... cripes.

they "should" be in a logical progression.  And since some of our
routines actually use the date/time stamp to sort and *validate*

on the file it just created won't match the time stamp on the file it
backed up.  The initial verification will fail.  Then the system will
generate a checksum on each file (ooohh... about 1.3 million files,
probably about 1/2 ofthem will get caught after the time change) to
compare to the original, and see it's no different.  So it adds that
filename to a log that gets emailed to me.
